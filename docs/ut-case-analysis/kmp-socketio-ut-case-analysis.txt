probe 时，先 ws 连接，成功后发送 Ping，收到 Pong 后，pause polling，再发送 Upgrade 包，发送成功（EVENT_DRAIN）后把 EngineSocket 的 transport 切换为 ws。

出问题时：
   ws 连接
   EngineSocket close，但要等 EngineSocket 的 EVENT_DRAIN（Disconnect 待发送）
   polling 发 Disconnect
   ws 连接成功
   ws 发 Ping
   ws 收到 Pong
   polling pause，但要等 poll 和 write (Disconnect)
   polling poll 完成，且不再继续 poll
   polling write 完成，触发 polling 的 EVENT_DRAIN，进而触发 EngineSocket 的 EVENT_DRAIN
   EngineSocket 正常处理 EVENT_DRAIN
   EngineSocket close 等到了 drain，但还要等 upgrade
   polling pause 完成
   监听 ws 的 EVENT_DRAIN，用于切换 transport
   ws 发 Upgrade
   ws 发 Ping 完成，触发 ws 的 EVENT_DRAIN
   把 EngineSocket 的 transport 切换为 ws
   ws 发 Upgrade 完成，再次触发 ws 的 EVENT_DRAIN，进而触发 EngineSocket 的 EVENT_DRAIN
   EngineSocket 处理 EVENT_DRAIN 时报错！

   *两个 ws 的 EVENT_DRAIN，不一定是 Ping 的先还是 Upgrade 的先，总之先监听（用于切换 transport），后来了两个 EVENT_DRAIN，第二个引发了问题。

问题关键在于 ws 的 Ping 和 Upgrade 会连续发送，而两次 EVENT_DRAIN 的顺序和「监听 ws 的 EVENT_DRAIN 用于切换 transport」的顺序无法保证。

   改成监听 Ping & Upgrade 的 drain 同时满足，也能解决此问题。
